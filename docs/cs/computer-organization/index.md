# 计算机组成原理

组成原理关注指令、处理器、存储层次和 I/O，是理解程序性能和系统瓶颈的基础。

## 子方向

- 数据表示。
- 指令系统。
- CPU 数据通路与控制。
- 流水线。
- Cache 和存储层次。
- I/O 与总线。

## 推荐学习路径

先理解数据表示和指令执行，再进入流水线、Cache 和内存层次，最后连接到程序性能分析。

## 笔记模板

- 硬件抽象。
- 执行流程。
- 性能指标。
- 局部性和瓶颈。
- 与代码行为的联系。
